## 本篇内容关键性思想

## 如何实现 readonly

readonly 和 reactive 在实现逻辑上都使用到了 proxy，不同的是，readonly 无需关心 set 实现，也无需拥有依赖收集和依赖触发的能力

## 重构代码带来哪些好处？

- 从项目的角度来说，重构代码更多的是让代码的层次清晰，同时提升代码的可复用性及可读性
- 从个人的角度来说，重构代码有助于提升程序员对代码的封装能力、以及逻辑思维能力，让代码可以更优雅。同时在阅读其他源码库的时候也能够得心应手
